Apa Itu **kwargs?

**kwargs adalah cara di Python untuk menerima banyak parameter dengan nama (keyword arguments) tanpa harus menentukan jumlahnya di awal.

Cara Kerja **kwargs

  • kwargs itu singkatan dari "keyword arguments".
  • ** (dua bintang) di depan kwargs berarti Python akan menangkap semua parameter yang dikirim dalam bentuk key=value dan menyimpannya dalam bentuk dictionary.

Contoh Sederhana

Misalnya aq bikin fungsi yang bisa menerima banyak parameter:

python
def cetak_info(**kwargs):
    for kunci, nilai in kwargs.items():
        print(f"{kunci}: {nilai}")

cetak_info(nama="Julia", umur=20, hobi="Menggambar")

Penjelasan:

  • nama="Julia", umur=20, dan hobi="Menggambar" dikirim ke fungsi.
  • **kwargs menangkapnya dalam bentuk dictionary: {"nama": "Julia", "umur": 20, "hobi": "Menggambar"}.
  • Lalu, kita pakai .items() untuk mengambil nama parameter (kunci) dan nilainya, lalu mencetaknya satu per satu.

Hasilnya:

makefile
nama: Julia
umur: 20
hobi: Menggambar

Kapan **kwargs Digunakan?

  • Saat ingin membuat fungsi yang fleksibel, bisa menerima parameter dengan nama tanpa harus menentukan jumlahnya.
  • Saat ingin mengolah data dalam bentuk pasangan key-value, misalnya dalam konfigurasi atau data JSON.

Kombinasi dengan *args

Kalau *args digunakan untuk menerima banyak nilai tanpa nama, **kwargs untuk menerima banyak nilai dengan nama. Bisa digabungkan:

python
def gabungan(*args, **kwargs):
    print("Positional arguments:", args)
    print("Keyword arguments:", kwargs)

gabungan(1, 2, 3, nama="Julia", hobi="Coding")

Hasilnya:

matlab
Positional arguments: (1, 2, 3)
Keyword arguments: {'nama': 'Julia', 'hobi': 'Coding'}

Jadi, args menangkap (1, 2, 3) sebagai tuple, sedangkan kwargs menangkap {"nama": "Julia", "hobi": "Coding"} sebagai dictionary.

3 Perbedaan > vs >=

Kita bandingkan dua kondisi ini:

Kondisi Salah (> saja)

python
if index_buku < 0 or index_buku > jumlah_buku:

Jika jumlah_buku = 3, maka batasnya adalah:

  • Indeks valid: 0, 1, 2
  • Tapi kondisi index_buku > jumlah_buku berarti index_buku > 3, yang mana tidak masuk akal karena indeks terakhir yang valid adalah 2.

Kesalahan: Kondisi ini masih mengizinkan index_buku = 3, padahal tidak valid!

Kondisi Benar (>=)

python
if index_buku < 0 or index_buku >= jumlah_buku:
  • Jika index_buku = 3, maka 3 >= 3 akan bernilai True, sehingga return False.
  • Jika index_buku = 2, maka 2 >= 3 akan bernilai False, sehingga tetap berjalan.

Kenapa harus pakai >=?
Karena jumlah_buku itu menunjukkan jumlah total data, bukan indeks terakhir.
Indeks terakhir selalu jumlah_buku - 1, jadi kalau index_buku >= jumlah_buku, itu pasti salah.

  • Pakai index_buku >= jumlah_buku, baru batasnya benar sehingga indeks 3 dianggap tidak valid.

Jadi, harus pakai >= supaya program tidak salah membaca indeks di luar batas.
